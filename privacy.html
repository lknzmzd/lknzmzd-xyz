Signals Privacy

The V2.7 subscription and email layer stores only the minimum information required to operate LKNZMZD Signals.

When the Cloudflare Worker backend is connected, the system stores email address, selected interests, subscription status, source page, consent version, timestamps, and optional non-reversible IP hash if configured. It does not need to store raw IP addresses.

Subscriber data, campaign logs, and delivery logs are stored in Supabase and are not exposed through public browser keys. Public writes are disabled; the Cloudflare Worker writes using a server-side service role key. If email sending is enabled, messages are sent through the configured email provider from the Worker.

Every broadcast email should include a tokenized unsubscribe link. To unsubscribe manually, use the self-service page at unsubscribe.html. You can also request manual removal by emailing ilkinazimzade@lknzmzd.com.
